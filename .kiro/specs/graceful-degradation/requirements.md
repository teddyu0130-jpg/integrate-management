# Requirements Document

## Project Description (Input)
Provider が応答しない場合に取得できたデータだけで表示を継続し、欠落属性を明示する障害許容の縮退表示。Provider の fetch 失敗はプロバイダー単位でキャッチし、エラーが他 Provider の表示をブロックしない。ユーザーには「App① のデータを取得できません」等の欠落明示を行いつつ、利用可能なデータで画面を成立させる。
